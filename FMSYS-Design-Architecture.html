01 — Overview

平台總覽

FMSYS Trading Suite Pro 是一款以 macOS 原生設計語言為基礎的桌面交易紀錄平台，專為專業交易者打造，整合交易日誌、回測分析、策略實驗室與投資組合管理四大核心模組。

shield_lock

安全認證

OAuth / Passkey 登入搭配 MFA 多因子驗證，確保帳戶安全

book_5

交易日誌

記錄每筆交易的進出場邏輯、情緒標籤與績效追蹤

analytics

回測分析

策略回測引擎，含權益曲線、勝率、Profit Factor 等指標

science

策略實驗室

開發、測試與優化演算法模型，含即時代碼編輯與參數調整

視窗行為細節 / Window Behaviors

Sidebar Collapse 動畫

寬度從 256px → 56px 的過渡行為

• 過渡動畫：width 300ms cubic-bezier(0.4, 0, 0.2, 1)
• 收合時：文字標籤淡出 (opacity 0, 150ms)，僅保留圖示
• 底部 Equity 卡片：收合時隱藏文字，僅顯示金額數字
• 手動觸發：Sidebar 頂部 Toggle 按鈕 或 ⌘ + \
• 自動觸發：視窗寬度縮至 Compact 斷點以下
• 展開時：文字標籤淡入 (opacity 1, 200ms, delay 100ms)

Inspector Panel 行為

右側面板在不同斷點下的顯示策略

• Full 模式：固定顯示，border-left 分隔
• Regular 模式：Overlay Drawer，從右側滑入
— 背景遮罩：rgba(0,0,0,0.3)
— 動畫：transform: translateX(100%) → 0, 300ms
— 關閉：點擊遮罩 或 Esc 鍵
• 觸發按鈕：Toolbar 區域 info 圖示
• 適用頁面：Strategy Lab (Inspector)、Journal (Properties)

錯誤狀態 / Error States

錯誤類型觸發場景UI 回饋操作選項
Login Failed 密碼錯誤 / 帳號不存在 輸入框邊框變為 #ff5f57，下方出現紅色錯誤訊息。輸入框輕微 shake 動畫 (300ms) 重新輸入 / Forgot Password
MFA Expired 驗證碼超時 (30s) OTP 輸入框清空，上方顯示 Warning Toast "Code expired"，Resend 連結脈衝高亮 Resend Code
MFA Invalid 驗證碼錯誤 (3次以上鎖定) 輸入框 shake 動畫，錯誤次數遞減提示 "2 attempts remaining" 重新輸入 / Contact Support

API Key 管理流程

API Key CRUD 流程

Create：Settings → API Keys tab → "Add New Key" → 選擇交易所 (Binance / Coinbase / Interactive Brokers 等) → 輸入 API Key + Secret → 選擇權限 (Read Only / Read + Trade) → Confirm → 顯示成功 Toast
Read：列表顯示已連接的 Key，僅顯示前 4 碼 + 遮罩 (••••••) + 交易所圖示 + 連線狀態 + 最後使用時間
Update：可修改 Label 名稱與權限層級，Secret 不可檢視或修改（需重新建立）
Delete：二次確認 Modal → 輸入 "DELETE" 確認文字 → 成功後 Toast + 列表更新

安全規範

• API Secret 僅在建立時顯示一次，之後永久遮罩
• 所有 Key 使用 AES-256-GCM 加密儲存
• 連線驗證：建立後自動 ping 交易所 API 確認有效性
• 異常偵測：連續 5 次 API 錯誤自動停用 Key 並通知
• 權限分離：Read Only Key 無法觸發任何交易操作
• 審計日誌：所有 Key 操作記錄至 Audit Log，含 IP + Timestamp
• Session 管理：MFA 驗證後 15 分鐘內可操作 Key，逾時需重新驗證

Session 與認證策略

項目規格
Session TTLAccess Token: 15 分鐘 / Refresh Token: 7 天 (Remember Me: 30 天)
MFA 觸發條件新裝置登入 / 30 天未使用的裝置 / 敏感操作 (API Key 管理、角色變更、帳戶刪除)
MFA 重試限制最多 5 次錯誤嘗試，之後鎖定 15 分鐘。3 次鎖定後需 Email 驗證解鎖
OAuth Session依循 OAuth 2.0 PKCE flow，Token 儲存於 Keychain (macOS)
併行登入最多 3 個同時活躍 Session，新登入踢出最舊 Session 並通知
Audit Log 保留90 天，Owner/Admin 可從 Settings → Security 檢視

關鍵架構決策 / Key Decisions

MVVM + Service Layer

View：純 SwiftUI 宣告式 UI，不含業務邏輯。使用 NavigationSplitView 實現三欄佈局，對應設計稿的 Sidebar + Main + Inspector 結構
ViewModel：使用 Swift 5.9 @Observable macro，取代舊版 ObservableObject。每個 Feature Module 一個 ViewModel，管理 UI 狀態與 Service 呼叫
Service：純商業邏輯層，不依賴 UI framework。負責 Repository 協調、資料轉換、驗證規則。可獨立單元測試
Repository：統一資料存取介面，內部決定從 SwiftData (local) 或 REST API (remote) 取資料。Offline-first 策略

Offline-First 離線優先策略

本機儲存：SwiftData 作為 single source of truth，所有讀取優先從本機載入
同步策略：背景 sync task，以 updated_at 時間戳比對差異，增量同步
衝突解決：Last-Write-Wins (LWW) 策略，伺服器為權威來源
離線操作：新增/編輯 Trade 時先寫入本機，標記為 pendingSync，網路恢復後自動上傳
優勢：即時回應、無網路也能完整使用 Journal 和 Strategy 編輯功能
